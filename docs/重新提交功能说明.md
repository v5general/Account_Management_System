# 重新提交功能说明

## 功能概述

当用户对被驳回的收支记录点击"重新提交"时，系统会：
1. **创建新记录**：生成一个全新的收支记录（状态为"待审核"）
2. **复制附件**：将原记录的所有附件转移到新记录
3. **删除旧记录**：软删除原来被驳回的记录（从列表中消失）

## 功能特点

### ✅ 优势

1. **审计清晰**：新记录有新的record_id，审计追踪更明确
2. **数据清洁**：避免数据库中存在大量状态为"已驳回"的记录
3. **附件保留**：用户无需重新上传附件
4. **用户友好**：用户只需修改需要调整的内容即可提交

### 🔄 工作流程

```
用户操作流程：
1. 用户在收支列表看到"已驳回"的记录
2. 点击"重新提交"按钮
3. 系统跳转到登记页面，自动填充原数据
4. 用户修改需要调整的内容
5. 点击"提交"按钮
6. 系统创建新记录 + 删除旧记录
7. 返回列表页，显示新的"待审核"记录
```

```
后端处理流程：
1. 接收重新提交请求
2. 验证旧记录状态为"已驳回"
3. 创建新记录（新record_id，状态=0）
4. 复制附件到新记录
5. 软删除旧记录（is_deleted=1）
6. 返回新记录ID
```

## 技术实现

### 后端API

**接口**：`PUT /api/v1/transactions/:id/resubmit`

**请求参数**：
```json
{
  "amount": 1000.00,
  "category_id": "xxx",
  "project_id": "xxx",
  "person_id": "xxx",
  "payment_method": "现金",
  "transaction_time": "2026-03-04 00:00:00",
  "remark": "修改后的备注"
}
```

**响应数据**：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "new_record_id": "record_new_xxx",
    "old_record_id": "record_old_xxx",
    "message": "重新提交成功，旧记录已删除"
  }
}
```

### 前端实现

#### 1. List.vue（收支列表）
- 按钮文本：**"重新提交"**（仅对已驳回记录显示）
- 点击后：跳转到登记页面并自动填充数据
- 存储：使用 `sessionStorage` 传递数据

#### 2. Income.vue / Expense.vue（登记页面）
- 检测：页面加载时检查 `sessionStorage` 中的重新提交数据
- 显示：提示消息"已加载被驳回的数据，请修改后重新提交"
- 提交：
  - 如果是重新提交，调用 `resubmitTransaction` API
  - 如果是新建，调用 `createTransaction` API
- 成功后：清除 `sessionStorage`，跳转回列表页

## 数据库变化

### 旧记录（被驳回）
```
record_id: record_old_xxx
status: 2 (已驳回)
is_deleted: 1 (已删除) ← 软删除，不在列表中显示
```

### 新记录（重新提交）
```
record_id: record_new_xxx (新的ID)
status: 0 (待审核)
is_deleted: 0 (未删除)
create_time: [当前时间]
creator_id: [当前用户]
```

### 附件表
```
attachment_id: xxx
record_id: record_new_xxx ← 更新为新记录ID
```

## 操作日志

系统会记录以下操作：
- 创建新记录：记录新的record_id
- 软删除旧记录：记录旧的record_id

## 注意事项

1. **附件转移**：附件是转移而非复制，旧记录删除后附件归属于新记录
2. **权限控制**：只有财务人员和管理员可以执行重新提交操作
3. **状态验证**：只能重新提交状态为"已驳回"(status=2)的记录
4. **数据完整性**：使用数据库事务确保操作的原子性

## 与原"复制"功能的区别

| 特性 | 原复制功能 | 新重新提交功能 |
|------|-----------|---------------|
| 记录数量 | 增加1条（旧记录保留） | 不变（删除旧记录，创建新记录） |
| record_id | 新记录有新ID | 新记录有新ID |
| 旧记录状态 | 保持"已驳回" | 软删除（不可见） |
| 附件处理 | 不复制附件 | 转移到新记录 |
| 审计追踪 | 需要关联新旧记录 | 新记录独立，旧记录已删除 |

## 示例场景

### 场景1：支出记录被驳回后重新提交

```
1. 财务人员登记支出：1000元，状态=待审核
2. 管理员审核后驳回：状态=已驳回，原因="凭证不清晰"
3. 财务人员点击"重新提交"
4. 系统自动填充原数据
5. 财务人员重新上传清晰的凭证
6. 提交后：
   - 旧记录（record_old_xxx）被软删除
   - 新记录（record_new_xxx）状态=待审核
   - 新凭证附件转移到新记录
```

### 场景2：收入记录被驳回后修改金额重新提交

```
1. 财务人员登记收入：5000元，状态=待审核
2. 管理员审核后驳回：状态=已驳回，原因="金额错误，应为4500元"
3. 财务人员点击"重新提交"
4. 系统自动填充原数据（5000元）
5. 财务人员修改金额为4500元
6. 提交后：
   - 旧记录（record_old_xxx）被软删除
   - 新记录（record_new_xxx）金额=4500元，状态=待审核
```

## 总结

重新提交功能实现了：
- ✅ 创建全新的记录（新record_id，待审核状态）
- ✅ 自动转移附件到新记录
- ✅ 删除旧的被驳回记录（软删除）
- ✅ 保持数据清洁，避免大量"已驳回"记录堆积
- ✅ 审计追踪清晰，每个新记录都有独立的ID和时间戳
