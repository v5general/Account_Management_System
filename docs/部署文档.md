## 2. 开发环境部署

### 2.1 克隆项目

```bash
git clone <repository-url>
cd Account_Management_System
```

### 2.2 数据库初始化

**1. 创建数据库**

```bash
mysql -u root -p
```

```sql
CREATE DATABASE IF NOT EXISTS account_management DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

**2. 导入表结构和初始数据**

```bash
mysql -u root -p account_management < database/schema.sql
mysql -u root -p account_management < database/init_data.sql
```

**3. 验证数据库**

```sql
USE account_management;
SHOW TABLES;
SELECT * FROM t_user;
```

### 2.3 后端配置

**1. 安装依赖**

```bash
cd backend
go mod download
```

**2. 配置文件**

编辑 `backend/config/config.yaml`：

```yaml
server:
  port: "8080"
  mode: debug  # 开发模式使用debug，生产环境使用release

database:
  host: localhost
  port: 3306
  database: account_management
  username: root
  password: your_password  # 修改为实际密码

jwt:
  secret: your_jwt_secret_key_change_in_production  # 生产环境请更换
  expire: 86400  # 24小时

oss:
  type: aliyun  # aliyun or tencent
  access_key: your_access_key
  secret_key: your_secret_key
  bucket: your_bucket
  region: your_region
```

**3. 启动后端服务**

```bash
# 开发模式
go run main.go

# 或编译后运行
go build -o account-management main.go
./account-management
```

### 2.4 前端配置

**1. 安装依赖**

```bash
cd frontend
npm install
```

**2. 配置代理**

`frontend/vite.config.ts` 已配置开发代理：

```typescript
server: {
```


**4. 访问系统**

打开浏览器访问：`http://localhost:3000`

默认账号：
- 用户名：`admin`
- 密码：`admin123`

## 3. 生产环境部署

### 3.1 后端部署

**1. 编译后端程序**

```bash
cd backend
# Linux/Mac
GOOS=linux GOARCH=amd64 go build -o account-management main.go

# Windows
go build -o account-management.exe main.go
```

**2. 创建部署目录**

```bash
sudo mkdir -p /opt/account-management
sudo mkdir -p /opt/account-management/uploads
sudo mkdir -p /opt/account-management/logs
sudo mkdir -p /opt/account-management/config
```

**3. 上传文件**

```bash
# 上传编译后的程序
scp account-management user@server:/opt/account-management/

# 上传配置文件
scp config/config.yaml user@server:/opt/account-management/config/
```

**4. 创建系统服务（Linux）**

创建 `/etc/systemd/system/account-management.service`：

```ini
[Unit]
Description=Account Management System
After=network.target mysql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/account-management
ExecStart=/opt/account-management/account-management
Restart=on-failure
RestartSec=5s

# 环境变量
Environment="GIN_MODE=release"

# 日志
StandardOutput=journal
StandardError=journal
SyslogIdentifier=account-management

[Install]
WantedBy=multi-user.target
```

**5. 启动服务**

```bash
# 重载systemd配置
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start account-management

# 设置开机自启
sudo systemctl enable account-management

# 查看服务状态
sudo systemctl status account-management

# 查看日志
sudo journalctl -u account-management -f
```

### 3.2 前端部署

**1. 构建前端项目**

```bash
cd frontend
npm run build
```

构建完成后，`dist` 目录包含静态文件。

**2. 配置 Nginx**

安装 Nginx：

```bash
sudo apt install nginx  # Ubuntu/Debian
sudo yum install nginx  # CentOS/RHEL
```

创建 Nginx 配置文件 `/etc/nginx/sites-available/account-management`：

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 修改为实际域名

    # 前端静态文件
    root /var/www/account-management;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_min_length 1000;

    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 上传文件代理
    location /uploads {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 文件上传大小限制
    client_max_body_size 100M;
}
```

**3. 部署静态文件**

```bash
# 创建目录
sudo mkdir -p /var/www/account-management

# 复制文件
sudo cp -r dist/* /var/www/account-management/

# 设置权限
sudo chown -R www-data:www-data /var/www/account-management
sudo chmod -R 755 /var/www/account-management
```

**4. 启用站点**

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/account-management /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 3.3 HTTPS 配置（可选）

使用 Let's Encrypt 免费 SSL 证书：

```bash
# 安装 certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

## 4. 数据库配置

### 4.1 创建专用数据库用户

```sql
-- 创建数据库用户
CREATE USER 'account_user'@'localhost' IDENTIFIED BY 'strong_password_here';

-- 授权
GRANT ALL PRIVILEGES ON account_management.* TO 'account_user'@'localhost';

-- 刷新权限
FLUSH PRIVILEGES;
```

### 4.2 数据库优化

**1. MySQL 配置优化**

编辑 `/etc/mysql/mysql.conf.d/mysqld.cnf`：

```ini
[mysqld]
# 字符集
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci

# 连接数
max_connections = 200

# 缓冲区
innodb_buffer_pool_size = 1G
key_buffer_size = 256M

# 日志
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow-query.log
long_query_time = 2
```

**2. 创建索引**

```sql
-- 收支流水表索引优化
ALTER TABLE t_transaction ADD INDEX idx_trans_time_category (transaction_time, category_id);
ALTER TABLE t_transaction ADD INDEX idx_trans_time_person (transaction_time, person_id);

-- 操作日志表索引优化
ALTER TABLE t_operation_log ADD INDEX idx_create_time_type (create_time, operation_type);
```

### 4.3 数据备份

**1. 创建备份脚本**

`/opt/backup/mysql-backup.sh`：

```bash
#!/bin/bash

# 配置
BACKUP_DIR="/opt/backup/mysql"
MYSQL_USER="account_user"
MYSQL_PASSWORD="your_password"
DATABASE="account_management"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u$MYSQL_USER -p$MYSQL_PASSWORD $DATABASE | gzip > $BACKUP_DIR/account_management_$DATE.sql.gz

# 删除30天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete

echo "Backup completed: account_management_$DATE.sql.gz"
```

**2. 设置定时任务**

```bash
# 添加到crontab
crontab -e

# 每天凌晨2点执行备份
0 2 * * * /opt/backup/mysql-backup.sh >> /opt/backup/mysql-backup.log 2>&1
```

```go
import "github.com/aliyun/aliyun-oss-go-sdk/oss"

type AliyunOSS struct {
    client *oss.Client
    bucket *oss.Bucket
}

func NewAliyunOSS(cfg *config.Config) *AliyunOSS {
    client, err := oss.New(cfg.OSS.Endpoint,
        cfg.OSS.AccessKey,
        cfg.OSS.SecretKey)
    if err != nil {
        log.Fatal(err)
    }

    bucket, err := client.Bucket(cfg.OSS.Bucket)
    if err != nil {
        log.Fatal(err)
    }

    return &AliyunOSS{client: client, bucket: bucket}
}
```

## 6. 常见问题

### 6.1 后端启动失败

**问题：** 端口被占用

```bash
# 查看端口占用
sudo lsof -i :8080

# 杀死进程
sudo kill -9 <PID>
```

**问题：** 数据库连接失败

- 检查数据库服务状态：`sudo systemctl status mysql`
- 验证配置文件中的数据库连接信息
- 检查防火墙规则

### 6.2 前端访问异常

**问题：** API 请求失败

- 检查后端服务是否正常运行
- 验证 Nginx 代理配置
- 查看浏览器控制台网络请求

**问题：** 路由刷新404

- 确认 Nginx 配置了 `try_files $uri $uri/ /index.html;`

### 6.3 文件上传失败

**问题：** 文件大小超限

- 修改 Nginx 配置：`client_max_body_size 100M;`
- 检查后端文件大小限制

**问题：** OSS 上传失败

- 验证 OSS 配置信息
- 检查存储空间是否充足
- 确认访问权限配置正确

### 6.4 性能优化

**1. 启用 Redis 缓存**

```bash
# 安装 Redis
sudo apt install redis-server

# 启动 Redis
sudo systemctl start redis
```

**2. 数据库连接池**

修改 `database/db.go`：

```go
db, err := gorm.Open(mysql.Open(dsn), &gorm.Config{
    Logger: logger.Default.LogMode(logger.Info),
    // 添加连接池配置
})
sqlDB, _ := db.DB()
sqlDB.SetMaxIdleConns(10)
sqlDB.SetMaxOpenConns(100)
sqlDB.SetConnMaxLifetime(time.Hour)
```
